---
description: Request Design/UX Review
---

1. **Context Setup**
   - Read `jobs-to-be-done/01-write-prd/reviewers/product_designer.md`.
   - Identify the target document provided by the user (or the active open file).

2. **Conduct Review**
   - Adopt the persona of the **Lead Product Designer**.
   - Review the document specifically for:
     - User journey clarity
     - Edge cases and error states
     - Consistency with design system principles
     - Accessibility and empathy
   - Ask: "How does the user *feel* here?"

3. **Output**
   - Provide the review in the chat.
